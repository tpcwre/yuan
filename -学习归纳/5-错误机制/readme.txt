1、写代码的时候难免有错误发生，怎么调。我们经常说好的代码是调试出来的。

    调试代码的技巧，方法：
        1、注意提示。 一般情况下，错误都会出现在提示的上面几行或下面几行。   
        2、调试代码的时候，使用断点调试法。我们会配合这exit来 进行处理
        3、输出调试法。

    PHP程序的错误类型：
        1、语法错误：最常见，也比较容易修复。会阻止脚本运行。第一个项目的时候最容易出现。
        2、运行时的错误；不会阻止PHP脚本运行，但是会阻止脚本的意图，输出一条错误，但PHP脚本继续执行。
        3、逻辑错误，这种错误，最麻烦。不阻止脚本运行。也不报错。

    为了减少这种问题的出现，所以我们的方法是做一点，调一点。 要养成勤按f5的习惯。


    PHP的错误级别：
        Notice：注意级别，没什么大碍，可以正常使用。（手破了皮）
        E_NOTICE

        WARNING:部分功能失效，脚本继续执行（左手骨折了，还有右手）
        E_WARNING


        ERROR：致命的运行错误。(脑袋掉了)
        E_ERROR


        E_ALL  所有的错误和警告

    设置PHP的错误级别
        方法1：在配置文件中进行修改(分号表示注释)
        error_reporting = 选项用来控制那些错误提示

        & 表示并且
        | 表示或者
        ~ 表示除了。

        注意：
            1、在修改配置文件的时候一定要将原配置文件备份一份出来。因为你很有可能改错喽。
            2、修改完配置文件后，一定要重新启动apache服务器

        方法2：在自己的脚本中修改（推荐的）

        error_reporting(E_ALL & ~E_NOTICE);

        注意：只在当前脚本（本页面）中有效。


        其他：
            1、php.ini中有一个display_errors选项，表示是否在浏览器中显示错误信息。
            display_errors = On

            2、ini_get(); 获得配置的信息
            3、ini_set('配置名','值'); 通过函数暂时的配置选项，在脚本结束时失效。
            4、ini_get_all();获得所有的配置信息（global_value表示的是php.ini中的设置，local_value表示的是本地的设置）


    日志管理
        我们将一个项目分为开发阶段和运行阶段。


        运营阶段会让用户看到错误提示吗？

        我们要达到，有错误，自己知道，但是不让用户看到


        让用户看到：
            1、用户体验不好。
            2、不安全。

        log_errors = On 是否开启日志记录
        error_log =  "C:\xampp\php\logs\php_error_log"  让错误日志存储到哪里。

        error_log = syslog  将错误日志存储到系统日志中。（不推荐）

        怎么查看系统日志： 我的电脑 -> 右键 -> 管理 -> 系统工具（事件查看器）
        -> 应用程序

        设置日志的步骤：
            1、将所有错误都显示处来。 error_reporting = E_ALL
            2、display_errors = Off 不显示错误报告。
            3、log_errors = on 开启错误日志
            4、error_log = '';

            syslog系统日志
