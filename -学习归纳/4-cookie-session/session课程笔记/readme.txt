会话控制。
    1、从逻辑上区分变量的级别
        1、当前页面的变量 ，只从声明处开始。一直到页面执行完成一直能够使用。
        2、两个页面之间传递变量 通过URL传值  ?NAME=YANHAIJING
        3、会话级别：同一个用户在一个网站内，共享属于自己的变量。
        4、全局变量：写如到数据库或文本中。

    2、会话控制：允许服务器跟踪同一个客户端连续做出的请求。

    3、cookie，服务器端给客户端的信息，让客户端进行存储。
    原理：一个用户在访问服务器时，服务器根据业务逻辑设置cookie给用户，用户在客户端电脑的特定的文件夹下面创建这个网站的文本文件（通常和域名有关系），用于记录要保存的这些变量。访问多个网站就创建多个这样的文件，用户在访问本网站的其他的页面的时候，http协议会将客户端设置的文本里面的内容带给其他的页面以供使用。

    这个保存网站信息的文件叫cookie文件。一整套，这种机制叫做cookie机制。

    文件的存储位置
        IE浏览器：C:\Documents and Settings\用户名\Cookies
        火狐浏览器：cookie不会存成一个文本。是加密过的。选项-》隐私-》使用自定义历史记录-》显示cookie

        其实在早期的时候cookie的使用是有争议的，服务器往客户端写文件是很敏感的。但是cookie其实是一个很特殊的东西，他必须在特定的文件夹下，特定的文件，纯纯的文本文件。

        1、保存cookie
        setcookie('名称','值','有效期');

        有效期：默认的时候为0，表示的是关掉浏览器就没有了。

        2、怎么取？
        所有客户端访问时自动将所有这个网站的cookie都带过来，带过来之后进入到一个全局的数组中去$_COOKIE

        像正常的数组一样使用他

        3、怎么删除
        setcookie('名称','',time()-1);  告诉客户端这个东西已经过期了。


        注意：
            1、设置cookie的值使用setcookie来设置。不能使用$_COOKIE['']=''来进行设置

            2、cookie里面可以设置数组，可以指定关联数组和索引数组，但是索引数组
            不指定下标的情况下，将会永远是0下标不会自动增长。

            3、第一次在设置cookie的页面下马上打印cookie的值是打印不出来的。

            4、注意，setcookie的第四个参数。是设置的cookie的路径，他默认的路径是当前目录，设置为/表示为根目录。你在哪里设置的cookie就要在哪里删。你设置了多少就删除多少。


            1）、路径是绝对路径还是相对路径  /113/20150728/ 
            2）、在下级目录中设置的cookie，在上级是不能用的。上级设置的cookie在下级是可以用的。


    4、session

        原理：客户端第一次登录网站的时服务器会给一个不重复的号码(session id)，
        客户端将session ID放在cookie里，在创建session id
        的同时会在服务器端创建一个文件（这个文件的文件名通常和session
        id的值是一样的），存放这个文件的文件夹也是已经指定好的。当下次客户通过网站的页面访问时session
        id
        通过cookie带过来，然后从存储session的文件夹里找对应的文件，读取处里面的值。

        1、开启session（只有开启了session才能告诉PHP调度服务器里面的一些session机制）
        session_start();

        session_start的作用：
            1、第一次会想客户端发送sessionid，并且创建session文件
            2、以后使用的时，根据http协议带过来的session id 值，找服务器里面的对应的文件。

            注意：你只要用session就必须使用session_start();

        2、如何使用session
            $_SESSION['下标名'] = 值

        3、删除session
            1、开启session_start();
            2、清空session数组 $_SESSION  unset();
            3、销毁服务器端的session文件 session_destroy();
            4、删除客户端的session id,删除客户端的session
            id必须在删除时候在setcookie的第四个参数上写上'/'
            setcookie(session_name(),'',time()-1,'/')

            session的这个名字现在是叫LIWANGYANG这个名字怎么来的？我能不能改变
            session_name();//获得服务器现在的session的名字
            session_id();//获得服务器现在的session id

        session有关的配置
           session.name 设置session的名称 
           session.save_path 设置保存的session文件的存储目录
