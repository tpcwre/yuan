    运算符

    小学的时候学过  + -  /

    计算机中的这些符号比数学中的要多。

    最早的时候计算机的运算符是按照运算元的个数来划分的。

    1 + 2 = 3

    1 和 2表示运算元
    + 就是运算符

    现代的计算机运算符是按照功能来进行划分的。
    分为：
        1、算数运算符
            +  加法   1 + 2 = ? 

                注意：
                    1、操作数中有字符串需要先将字符串转换为整数或浮点数.

            - 减法

            *
                注意：
                    1、乘法在计算机中使用*而不是x

            / 除法
                注意：
                    1、被除数/除数 = 商
                    2、0不能为除数

            % 求余(求模)
                注意：
                    1、求模的第一个数为负数那么他的结果就为负数。
                    2、第一个数字小于第二个数，余数为第一个数
                    3、第二个数不能为0，否则会报错。

            ++ 累加1
                自增
                ++可以放在运算数（变量）的前面也可意识后面，但是意义是不同的.

                放前面和放后面的区别：
                    放在运算数的后面先用后加。
                    放在运算数的前面先加后用。

                注意：
                    1、$a=$a +1 和 $a++是不同的；
                    2、当$a为布尔型的时候那么布尔型的值不会参加自增/自减运算。会等于原来的值

            -- 自减
                放在前面或后面

                区别和++的区别是一样的。

        2、赋值运算符
            = 赋值运算符   将等号右边的值或者是表达式给变量。

            +=  $a += 1   $a = $a + 1;
            -=
            *=
            /=  $a /= 1  $a = $a / 1;
            %=

            . 字符串连接符

            注意：
                1、直接连接的时候可以连接变量、常量、字符串
                2、连接的变量可以是任何的标量。
                3、整数和浮点数连接时原样连接
                4、布尔型true先转换为1，再连接。false转换为空字符串再连接

            .=  $a .= 'a'  $a = $a . 'a';

        3、比较运算符
            18  ->  19   大一岁。

            我今年18岁。马尧今年19岁。 
            
            马尧比我小。 这句话是假的。

            比较运算符 得到的结果应该是布尔型的值。 true或false

            > 大于
            < 小于
            >= 
            <=

            == 等于
                注意：
                    1、一个等号是赋值，两个等号是判断是否等于
                    2、PHP弱类型语言，可以让不同类型的值做比较（先转换为同以类型再比较）.

            === 全等于
                注意：不仅内容要相等，而且两个内容的类型也要相等才返回true。

            != 不等于
            !== 不全等于


            一个等号是赋值，两个等号是判断是否相等，三个等号是判断是否全等。

            != 不等于     apple != banana
            !== 不全等于   因为我要去进行比较类型

        4、逻辑运算符

            比较运算符只能进行一个比较 ，有时候你需要多个条件进行比较
            所以我们需要逻辑运算符  将多个条件进行连接

            逻辑运算符得到的结构也是布尔型

            !逻辑非  取反。  操作数为true时返回false，操作数为false时返回true    马尧  现在是男的。  !true 

            && ：逻辑与  两边操作数都为true时候返回true否则返回false

            || ：逻辑或 两边的操作数只要有一个为true时候就返回真两个都为false的时候才返回false

            注意：
                1、逻辑运算符经常和比较运算符综合起来使用，形成一个大的条件。
                2、短路
                && 如果第一个表达式返回false将不会检查第二个表达式。
                || 如果第一个表达式返回true将不检查第二个表达式。

        5、位运算符
            & 位与 ：两个操作数都为1的位设置为1
            | 位或 ： 有任何一个相对应的位为1的就设置为1

            做之前先转换为2进制，然后比对位，得出结果后也是二进制的，但是你要转回来。


        6、字符串运算符
            .  .=
        7、其他运算符
        ? : 三元运算符，提供简单的判断 
        格式： 运算元1 ? 运算元2 : 运算元3;

        运算元1为条件 ，运算元2为表达式，运算元3表达式。

        如果说运算元1为真那么将运算元2的东西返回，如果运算元1为假那么将运算元3的东西返回。

        注意：
            1、表达式只能放置一条语句。
            2、表达式的值可以返回，所以可以进行赋值操作

        `` 反引号，执行运算符 得到的结果将是系统真实运行这个命令时返回的东西。
        （不建议你用）  

        @ 错误抑制符
            抑制  ->  
        只是临时屏蔽错误，不影响程序的执行。但是错误依然存在。

        8、运算符的优先级
        在数学里面先算乘除后算加减。如果要改变优先级使用括号。

        程序里面是从上至下，从左至右执行的，他对于运算符也有优先级，但是你用小学数学的只是就可以了（加括号）

        1、让多个表达式看起来是一个整体。
        2、提高优先级

    分支结构：
        不管你的项目有多大，有多复杂，分支结构就这几种情况

        单一分支结构（if）
        格式：
            if(表达式){

            }


            如果 马尧是男的  就干什么

        表达式：
        最精简的定义就是任何有值的东西就是表达式。将数据用运算符按照对应的规则连接起来的式子就叫表达式

        如果表达式为true时就执行括号内的内容，表达式为false时将忽略括号内的内容，继续向下执行

        注意：
            1、可以在括号中直接指定true或false，但是没有什么实际的意义。
            2、如果大括号内只有一条语句那么可以省略大括号（建议你不要省略）

        双向分支结构(else从句)
        格式：
            if(表达式){
                成立的时候执行
            }else{
                不成立的时候执行
            }


            如果 马尧是男的  就干什么 如果不是 那就好办了。

        如果表达式为true就执行if大括号内的语句，条件表达式为false就执行else大括号内的内容。

        注意：
            1、else必须和if语句搭配使用。
            2、你也可以使用
                if(表达式)
                    单条的语句
                else
                    单条的语句

                不用加括号   建议你加上。

            3、程序中如果有if和else必须执行一个括号中的代码。

        代码是有先后顺序的。每个大段的代码都是由小段的代码组成的。你再写的时候一定要知道整个的思路然后再去写。
